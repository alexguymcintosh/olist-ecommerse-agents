## Vision

### 10x Check
A live AI Club dashboard where each student runs their own agent system. A shared leaderboard shows whose agents produced the most accurate, novel, or actionable insights. The Connector Agent writes a weekly "CEO briefing" for a fictional Olist executive. Students present their Connector outputs and defend them.

> **Note:** The 10x Check is aspirational only — NOT in MVP scope. It describes a compelling future direction. None of the dashboard, leaderboard, or cadence features are deliverables for this plan.

### Platonic Ideal
A new student clones the repo, runs `python run_all.py`, and gets a beautiful terminal output:

```
OLIST BUSINESS BRIEFING — April 2018
=====================================
CUSTOMER (96k customers, ~100k orders): 0% repeat rate. Top risk: delivery >4 weeks drops NPS by 50%.
PRODUCT (73 cats): Health_beauty leads at $1.26M. Top opportunity: watches_gifts momentum.
SELLER (3095 sellers): SP concentrated. ~60% of revenue from top sellers. Risk: HHI rising.
CONNECTOR: Cross-signal — delivery delay is the single biggest lever. Fix logistics, repeat rate follows.
```

Then they read the code and understand exactly how each number was produced. That's the teaching moment.

## Architecture

### Build Order (explicit)
1. `utils/schema.py` — TypedDicts (no runtime deps)
2. `utils/validators.py` — validate_output() using schema
3. Domain agents (Customer, Product, Seller) — each independently testable
4. `tests/` — write alongside each agent
5. `run_all.py` — orchestrator, wire everything
6. `ConnectorAgent` — last, after all domain agents pass tests
7. `TODOS.md` — capture deferred items

### Approach Selected
**Class-based agents with shared BaseAgent + `run() -> DomainAgentOutput` (Approach B + BaseAgent)**

**BaseAgent (utils/base_agent.py)** — holds all shared logic:
- `run(self) -> DomainAgentOutput` — orchestrates: prepare_data → compute_metrics → build_prompt → call_llm → parse → validate → return
- `_parse_llm_response(raw: str) -> dict` — strips code fences, json.loads, raises ValueError on failure
- `_handle_error(e: Exception) -> DomainAgentOutput` — returns partial output with `risk_flags: ["agent_failed"]`
- `__init__(self, data: dict, model: str = RND_MODEL, teach: bool = False)`

Each domain agent (CustomerAgent, ProductAgent, SellerAgent) subclasses BaseAgent and implements:
- `_prepare_data(self) -> pd.DataFrame` — joins the relevant tables (see Join Contracts below)
- `_compute_metrics(self, df: pd.DataFrame) -> dict` — deterministic pandas aggregations on FULL df (no sampling)
- `_build_question(self, metrics: dict, sample_str: str) -> str` — LLM question string using pre-computed metrics

ConnectorAgent (does NOT subclass BaseAgent — different interface):
- `__init__(self, outputs: list[DomainAgentOutput], model: str = RND_MODEL, teach: bool = False)`
- `run(self) -> ConnectorOutput` — synthesises cross-domain insights from structured outputs
- Minimum viable input: 1 domain agent output. If 0 outputs, raises `ValueError("No domain agent outputs to synthesise")`.
- If all outputs have `risk_flags: ["agent_failed"]`: still runs, briefing notes all agents failed.
- If some outputs have `risk_flags: ["agent_failed"]`: notes failed agents, synthesises from available.
- Deterministic fallback: if own LLM call fails, renders a plain text summary from domain outputs without synthesis.

### Metric Computation Rule (HARD RULE)
**Pandas computes ALL numeric metrics from the FULL (unsampled) DataFrame.**
**LLM receives pre-computed metric dict + a sample of rows for narrative context only.**

```python
# CORRECT pattern in each domain agent:
df = self._prepare_data()                            # full joined df
metrics = self._compute_metrics(df)                  # deterministic, on full df
sample_str = data_loader.to_llm_string(              # sample for narrative context
    data_loader.sample(df, n=50), max_rows=50)
question = self._build_question(metrics, sample_str) # LLM gets pre-computed facts

# WRONG pattern (never do this):
# metrics = json.loads(query_llm("compute repeat_rate from these rows: " + sample_str))
```

### Join Contracts (per domain agent)

**CustomerAgent:**
```
Tables: orders + customers + reviews + payments
Grain: one row per order
Join: orders.merge(customers, on=customer_id).merge(reviews, on=order_id, how=left).merge(payments, on=order_id, how=left)
Key metrics computed: repeat_rate, avg_review_score, avg_delivery_days, pct_late_delivery, top_payment_type
```

**ProductAgent:**
```
Tables: order_items + products + categories
Grain: one row per order_item
Join: order_items.merge(products, on=product_id).merge(categories, on=product_category_name, how=left)
Key metrics computed: top_category, top_category_revenue, total_categories, avg_order_value
```

**SellerAgent:**
```
Tables: sellers + order_items + reviews
Grain: one row per order_item with seller
Join: order_items.merge(sellers, on=seller_id).merge(reviews, on=order_id, how=left)
Key metrics computed: total_sellers, pct_sao_paulo, top_seller_revenue_concentration (Pareto: top-20% share)
```

Note: `data_loader.py` already has `load_all()`, `sample()`, and `to_llm_string()` (verified in existing code).

### LLM Prompt Template
Each domain agent follows this structure:
```python
messages = build_analyst_prompt(
    data_summary=data_loader.to_llm_string(sampled_df),
    question="""Analyse this data and return a JSON object with exactly these keys:
{
  "insights": ["<3-5 concise bullet findings as plain English strings>"],
  "metrics": {<agent-specific metrics — see schema.py for required keys>},
  "top_opportunity": "<one sentence>",
  "risk_flags": ["<0+ short strings for Connector to act on>"]
}
Return ONLY the JSON object. No prose, no markdown.
"""
)
```

### JSON Parsing Contract
Implementation uses **brace-window extraction**: take the substring from the first `{` through the last `}` in the raw LLM string, then `json.loads` that substring. This tolerates leading prose, trailing text, and typical `` ```json `` fences without relying on exact fence stripping order. (A naive strip-then-`loads` on the whole string is more brittle when the model adds commentary.)

```python
import json

raw = query_llm(messages, model=self.model)
s = raw.strip()
start, end = s.find("{"), s.rfind("}")
json_str = s[start : end + 1]
try:
    parsed = json.loads(json_str)
except json.JSONDecodeError as e:
    raise ValueError(f"{self.AGENT_NAME}: LLM returned non-JSON response: {e}\nRaw: {raw[:200]}")
```
If parsing fails, the outer LLM error handler wraps it and returns a partial output with `risk_flags: ["agent_failed"]`.

### Output Contract (utils/schema.py)
```python
# utils/schema.py — zero runtime dependencies, no imports from other utils
from typing import TypedDict

class CustomerMetrics(TypedDict):
    repeat_rate: float           # 0.0–1.0
    avg_review_score: float      # 1.0–5.0
    avg_delivery_days: float
    pct_late_delivery: float     # 0.0–1.0
    top_payment_type: str

class ProductMetrics(TypedDict):
    top_category: str
    top_category_revenue: float
    total_categories: int
    avg_order_value: float

class SellerMetrics(TypedDict):
    total_sellers: int
    pct_sao_paulo: float              # 0.0–1.0
    top_seller_revenue_concentration: float  # revenue share held by top sellers (Pareto)

class DomainAgentOutput(TypedDict):
    agent: str            # "customer" | "product" | "seller"
    timestamp: str        # ISO 8601
    insights: list[str]   # 3-5 human-readable English bullet findings
    metrics: dict         # CustomerMetrics | ProductMetrics | SellerMetrics (agent-specific)
    top_opportunity: str  # one-line business recommendation (human-readable)
    risk_flags: list[str] # short machine-readable strings for Connector: e.g., "late_delivery_high", "agent_failed"

class PriorityAction(TypedDict):
    action: str    # e.g., "Reduce delivery SLA to <7 days"
    agent: str     # "customer" | "product" | "seller" | "cross-domain"
    urgency: str   # "HIGH" | "MEDIUM" | "LOW"

class ConnectorOutput(TypedDict):
    timestamp: str
    cross_domain_insights: list[str]  # 3-5 human-readable cross-domain findings
    strategic_recommendation: str     # top-priority one-liner
    priority_actions: list[PriorityAction]  # e.g., [{"action": "Fix delivery", "agent": "seller", "urgency": "HIGH"}]
    briefing: str  # full narrative string for Rich terminal display
```

Note on `metrics` typing: The field is typed as `dict` at the `DomainAgentOutput` level because different agents return different metric shapes. `validate_output()` enforces the per-agent required keys at runtime. TypedDict static checking is a future improvement (deferred).

### validate_output() Specification (utils/validators.py)
```python
REQUIRED_KEYS = {"agent", "timestamp", "insights", "metrics", "top_opportunity", "risk_flags"}

REQUIRED_METRICS = {
    "customer": {"repeat_rate", "avg_review_score", "avg_delivery_days", "pct_late_delivery", "top_payment_type"},
    "product":  {"top_category", "top_category_revenue", "total_categories", "avg_order_value"},
    "seller":   {"total_sellers", "pct_sao_paulo", "top_seller_revenue_concentration"},
}

def validate_output(output: dict, agent_name: str) -> None:
    """Raises ValueError with a clear message if output is malformed."""
    missing = REQUIRED_KEYS - set(output.keys())
    if missing:
        raise ValueError(
            f"ValidationError: {agent_name}.run() missing required keys: {missing}\n"
            f"Expected: {REQUIRED_KEYS}\nGot: {set(output.keys())}"
        )
    agent_type = output.get("agent")
    if agent_type in REQUIRED_METRICS:
        metrics_missing = REQUIRED_METRICS[agent_type] - set(output["metrics"].keys())
        if metrics_missing:
            raise ValueError(
                f"ValidationError: {agent_name} metrics missing keys: {metrics_missing}"
            )
```

### LLM Error Handling Contract
```python
try:
    response = query_llm(messages, model=self.model)
    parsed = json.loads(...)  # per JSON parsing contract above
    output = DomainAgentOutput(agent=self.AGENT_NAME, timestamp=..., **parsed)
    validate_output(output, self.__class__.__name__)
    return output
except Exception as e:
    return DomainAgentOutput(
        agent=self.AGENT_NAME,
        timestamp=datetime.utcnow().isoformat(),
        insights=[f"LLM call failed: {e}"],
        metrics={},
        top_opportunity="unavailable",
        risk_flags=["agent_failed"]
    )
```

## Clarifications for Implementers

**`build_analyst_prompt()`:** This already exists in `utils/openrouter_client.py`. Signature: `build_analyst_prompt(data_summary: str, question: str) -> list[dict]`. Each agent calls it directly — no need to re-implement.

**`data_loader.load_all()`:** `run_all.py` calls `data_loader.load_all()` which returns `{"orders": df, "customers": df, "order_items": df, "products": df, "sellers": df, "payments": df, "reviews": df, "categories": df}`. Pass the full dict to each agent constructor — each agent selects its relevant DataFrames internally.

**`AGENT_NAME`:** Each agent class defines a class-level constant: `AGENT_NAME: str = "customer"` (or "product" / "seller" / "connector"). Used in error handling and in the `agent` field of the output.

**ValidationError vs LLM failure:** Catch `ValueError` (from `validate_output()`) separately before the broad `except Exception`, so students see the exact validation message rather than a generic `agent_failed` flag.

**`--teach` data flow:** Implement via `teach: bool = False` constructor parameter. When `True`, each `run()` call prints prompt + raw response to Rich console internally, then returns the structured output normally. `run_all.py` passes `teach=True` when `--teach` flag is set.

**datetime deprecation (Python 3.12+):** Use `datetime.now(timezone.utc).isoformat()` throughout, not `datetime.utcnow()`.
